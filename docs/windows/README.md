# CodexBar for Windows — Refactor Docs

This folder is the working spec for turning `steipete/CodexBar` (macOS-only Swift / AppKit) into a first-class Windows 10/11 system-tray app with full feature parity.

It collects:

1. What the Mac app actually does and where it touches macOS-only APIs.
2. What the existing community Windows forks have tried, and how far they got.
3. The stack options available on Windows, with concrete trade-offs.
4. The recommended architecture and why.
5. The Windows-native UX spec — how it should *feel* to a Windows user.
6. A phased roadmap.
7. Risks and open decisions.

## TL;DR

- The Mac app is **~250 Swift files**: AppKit status item + SwiftUI prefs + a `CodexBarCore` library that does all the provider polling, parsing, OAuth/cookie/CLI plumbing, and cost scanning.
- The platform-dependent surface is **smaller than it looks**. The provider logic is mostly HTTP + JSON + filesystem reads. The Mac-only parts cluster around: `NSStatusBar` (tray), `NSPopover`/AppKit menu (popup), `Security.framework` Keychain, Sparkle (auto-update), Safari `binarycookies`, `~/Library/...` cookie paths, KeyboardShortcuts (sindresorhus), Vortex (confetti).
- There are **4 community Windows attempts**. Exactly one is healthy: [`Finesssee/Win-CodexBar`](https://github.com/Finesssee/Win-CodexBar) — Tauri + React + a shared Rust crate, ~392★, MIT, active May 2026, 40 providers ported, dynamic two-bar tray icon, DPAPI-backed secret storage, Inno Setup installer, WSLg path. The others are dead, stubs, or wrap-the-CLI.
- The recommended path is **not** "port Swift to Windows." It is: **adopt Win-CodexBar’s Tauri + Rust architecture as the base** (either by re-forking from it or by porting its Rust core into this repo), and keep parity with upstream `steipete/CodexBar` by re-implementing each provider against the same external contracts (cookies, OAuth, CLI output, log files).
- Why: Swift on Windows is viable but immature; AppKit/`NSStatusBar`/`NSPopover` have no real Windows equivalent that produces a native feel; the bulk of the value is in the provider plumbing, which Win-CodexBar has already ported and which we’d otherwise rewrite from scratch. Rust + Tauri is a proven, cheap, fast tray-app stack on Windows and is what every healthy fork or comparable app has converged on.

See [04-recommended-architecture.md](04-recommended-architecture.md) for the actual recommendation and a back-out plan if you’d rather not depend on a downstream community port.
